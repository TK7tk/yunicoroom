# GitHub Advanced Tips & Tricks

## Workflow & Productivity
- **Keyboard Shortcuts**:
    - `t`: Fuzzy file finder in repo.
    - `y`: Permalink to specific line (canonical link).
    - `Shift` + Click: Select range of lines.
    - `.` (dot): Open repo in github.dev (VS Code in browser).
- **Command Palette**: `Ctrl` + `k` (or `Cmd` + `k`) to run commands from anywhere.

## Repository Management
- **Templates**: PR templates and Issue templates to standardize input.
- **Git LFS**: Large File Storage for binaries/assets.
- **Naming**: Clear conventions (kebab-case preferred often).
- **Splitting**: Use submodules/subtrees for massive monorepos.

## Collaboration
- **Branch Protection**: Enforce reviews, CI checks before merge.
- **Code Owners**: Auto-assign reviews based on file paths.
- **Draft PRs**: Open early for feedback without triggering full review/merge expectation.

## "Secret" Tools (URL Hacks)
- `github1s.com`: View repo in VS Code reader mode.
- `gitdiagram.com`: Visualize repo structure.
- `gitsummarize.com`: Summary of repo purpose/activity.
